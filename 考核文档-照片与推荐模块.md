## 一、照片管理模块（CRUD）

### 1.1 功能概述

照片管理模块负责用户上传照片的增删改查操作，支持按用户ID、状态进行筛选查询，包含分页功能。

### 1.2 涉及数据表

| 表名 | 说明 | 角色 |
|-----|------|-----|
| `photo` | 照片表 | 主表 |
| `user` | 用户表 | 关联表 |

### 1.3 表结构

**photo 表（照片表）**：

```sql
CREATE TABLE photo (
    PhotoID INT PRIMARY KEY AUTO_INCREMENT,
    Photo_UserID INT,
    PhotoURL VARCHAR(500),
    UploadTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    AnalysisStatus VARCHAR(20) DEFAULT '未分析',
    FOREIGN KEY (Photo_UserID) REFERENCES user(UserID)
);
```

**user 表（用户表）**：

```sql
CREATE TABLE user (
    UserID INT PRIMARY KEY AUTO_INCREMENT,
    NickName VARCHAR(50),
    Phone VARCHAR(20),
    Gender VARCHAR(10),
    RegisterTime DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 1.4 表关联关系

```
┌─────────────┐        Photo_UserID        ┌─────────────┐
│   photo     │◄───────────────────────────│    user     │
│─────────────│                            │─────────────│
│ PhotoID (PK)│                            │ UserID (PK) │
│ Photo_UserID│───外键───►                 │ NickName    │
│ PhotoURL    │                            │ Phone       │
│ UploadTime  │                            │ Gender      │
│ AnalysisStatus│                          │ RegisterTime│
└─────────────┘                            └─────────────┘
```

**关联说明**：
- `photo.Photo_UserID` → `user.UserID`（外键关联）
- 一个用户可以上传多张照片（一对多关系）

### 1.5 API接口

| 接口 | 方法 | 功能 |
|-----|------|-----|
| `/photo/page` | GET | 分页查询照片列表 |
| `/photo/add` | POST | 新增照片记录 |
| `/photo/update` | POST | 修改照片信息 |
| `/photo/delete` | POST | 删除照片记录 |

### 1.6 多表联查说明

照片列表查询使用 **Photo 表** 和 **User 表** 的左连接：

```sql
SELECT p.PhotoID, p.Photo_UserID as UserID, u.NickName as UserName, 
       p.PhotoURL as ImageURL, p.UploadTime, p.AnalysisStatus as Status
FROM photo p 
LEFT JOIN user u ON p.Photo_UserID = u.UserID 
WHERE 1=1
ORDER BY p.UploadTime DESC
LIMIT ? OFFSET ?
```

**联查目的**：将照片关联到用户，显示上传照片的用户昵称。

## 二、分析结果查询模块（复杂查询）

### 2.1 功能概述

分析结果查询模块展示AI对用户照片的分析结果，支持按用户昵称、肤色、脸型进行多条件筛选查询，包含排序功能。

### 2.2 涉及数据表

| 表名 | 说明 | 角色 |
|-----|------|-----|
| `analysisresult` | 分析结果表 | 主表 |
| `photo` | 照片表 | 关联表 |
| `user` | 用户表 | 关联表 |
| `style` | 风格表 | 统计关联表 |
| `stylematch` | 风格匹配表 | 统计关联表 |

### 2.3 表结构

**analysisresult 表（分析结果表）**：

```sql
CREATE TABLE analysisresult (
    AnalysisID INT PRIMARY KEY AUTO_INCREMENT,
    Analysis_PhotoID INT,
    SkinTone VARCHAR(20),
    FaceShape VARCHAR(20),
    BodyShape VARCHAR(20),
    AnalysisTime DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (Analysis_PhotoID) REFERENCES photo(PhotoID)
);
```

**style 表（风格表）**：

```sql
CREATE TABLE style (
    StyleID INT PRIMARY KEY AUTO_INCREMENT,
    StyleName VARCHAR(50),
    StyleImageURL VARCHAR(500),
    StyleDescription TEXT
);
```

**stylematch 表（风格匹配表）**：

```sql
CREATE TABLE stylematch (
    SM_ID INT PRIMARY KEY AUTO_INCREMENT,
    SM_AnalysisID INT,
    SM_StyleID INT,
    FOREIGN KEY (SM_AnalysisID) REFERENCES analysisresult(AnalysisID),
    FOREIGN KEY (SM_StyleID) REFERENCES style(StyleID)
);
```

### 2.4 表关联关系

```
┌──────────────────┐      Analysis_PhotoID      ┌─────────────┐      Photo_UserID      ┌─────────────┐
│  analysisresult  │◄──────────────────────────│    photo    │◄──────────────────────│    user     │
│──────────────────│                            │─────────────│                         │─────────────│
│ AnalysisID (PK)  │                            │ PhotoID (PK)│                         │ UserID (PK) │
│ Analysis_PhotoID │───外键───►                 │ PhotoURL    │                         │ NickName    │
│ SkinTone         │                            │ UploadTime  │                         │ Phone       │
│ FaceShape        │                            └─────────────┘                         └─────────────┘
│ BodyShape        │
│ AnalysisTime     │
└────────┬─────────┘
         │ SM_AnalysisID
         ▼
┌──────────────────┐
│   stylematch     │
│──────────────────│
│ SM_ID (PK)       │
│ SM_AnalysisID    │───外键───►
│ SM_StyleID       │───外键───►
└────────┬─────────┘
         │ SM_StyleID
         ▼
┌─────────────┐
│    style    │
│─────────────│
│ StyleID (PK)│
│ StyleName   │
│ StyleImageURL│
└─────────────┘
```

**关联说明**：
- `analysisresult.Analysis_PhotoID` → `photo.PhotoID`（一对多关系）
- `photo.Photo_UserID` → `user.UserID`（一对多关系）
- `stylematch.SM_AnalysisID` → `analysisresult.AnalysisID`（一对多关系）
- `stylematch.SM_StyleID` → `style.StyleID`（一对多关系）

### 2.5 API接口

| 接口 | 方法 | 功能 |
|-----|------|-----|
| `/analysis/list` | GET | 多条件查询分析结果列表 |
| `/style/rank` | GET | 风格匹配排行统计 |

### 2.6 多表联查说明（分析结果查询）

分析结果查询使用 **AnalysisResult、Photo、User** 三张表的左连接：

```sql
SELECT a.AnalysisID, u.UserID, u.NickName as UserName, p.PhotoURL, 
       a.SkinTone, a.FaceShape, a.BodyShape, a.AnalysisTime
FROM analysisresult a 
LEFT JOIN photo p ON a.Analysis_PhotoID = p.PhotoID 
LEFT JOIN user u ON p.Photo_UserID = u.UserID 
WHERE 1=1
ORDER BY a.AnalysisTime DESC
LIMIT ? OFFSET ?
```

**联查目的**：将分析结果关联到照片和用户，完整展示分析上下文（用户昵称、照片地址、肤色、脸型、体型）。

### 2.7 多表联查说明（风格匹配排行）

风格匹配排行使用 **Style、StyleMatch** 两张表的左连接：

```sql
SELECT s.StyleID, s.StyleName, s.StyleImageURL,
       COUNT(sm.SM_AnalysisID) as MatchCount,
       ROUND(AVG(CASE WHEN sm.SM_StyleID IS NOT NULL THEN 1 ELSE 0 END) * 100, 1) as AvgScore
FROM style s
LEFT JOIN stylematch sm ON s.StyleID = sm.SM_StyleID
GROUP BY s.StyleID, s.StyleName
ORDER BY MatchCount DESC
```

**联查目的**：统计每个风格被匹配的次数，生成风格匹配排行榜。

## 三、推荐记录管理模块（CRUD）

### 3.1 功能概述

推荐记录管理模块负责管理系统生成的推荐记录，支持增删改查操作，包含推荐类型、状态筛选功能。

### 3.2 涉及数据表

| 表名 | 说明 | 角色 |
|-----|------|-----|
| `recommendation` | 推荐记录表 | 主表 |
| `user` | 用户表 | 关联表 |
| `analysisresult` | 分析结果表 | 关联表 |

### 3.3 表结构

**recommendation 表（推荐记录表）**：

```sql
CREATE TABLE recommendation (
```

## 四、推荐详情查询模块（复杂查询）

### 4.1 功能概述

推荐详情查询模块展示推荐记录的完整信息，包括关联的分析结果数据，支持多条件筛选。

### 4.2 涉及数据表

| 表名 | 说明 | 角色 |
|-----|------|-----|
| `recommendation` | 推荐记录表 | 主表 |
| `user` | 用户表 | 关联表 |
| `analysisresult` | 分析结果表 | 关联表 |

### 4.3 表关联关系

```
┌──────────────────┐      Rec_UserID      ┌─────────────┐
│ recommendation   │◄────────────────────│    user     │
│──────────────────│                      │─────────────│
│ RecommendationID │                      │ UserID (PK) │
│ (PK)             │───外键───►           │ NickName    │
│ Rec_UserID       │                      └─────────────┘
│ Rec_AnalysisID   │
│ RecommendationType│
│ RecommendationReason│
│ IsAccepted       │
│ RecommendationTime│
└────────┬─────────┘
         │ Rec_AnalysisID
         ▼
┌──────────────────┐
│  analysisresult  │
│──────────────────│
│ AnalysisID (PK)  │
│ SkinTone         │
│ FaceShape        │
│ BodyShape        │
│ AnalysisTime     │
└──────────────────┘
```

**关联说明**：
- `recommendation.Rec_UserID` → `user.UserID`（一对多关系）
- `recommendation.Rec_AnalysisID` → `analysisresult.AnalysisID`（一对多关系）

### 4.4 多表联查说明

推荐详情查询使用 **Recommendation、User、AnalysisResult** 三张表的左连接：

```sql
SELECT r.RecommendationID as RecommendID, u.NickName as UserName,
       a.SkinTone, a.FaceShape,
       r.RecommendationType as RecommendType, r.RecommendationReason as RecommendReason,
       r.IsAccepted, r.RecommendationTime as RecommendTime
FROM recommendation r 
LEFT JOIN user u ON r.Rec_UserID = u.UserID 
LEFT JOIN analysisresult a ON r.Rec_AnalysisID = a.AnalysisID 
WHERE 1=1
ORDER BY r.RecommendationTime DESC
LIMIT ? OFFSET ?
```

**联查目的**：将推荐记录关联到用户和分析结果，展示用户的肤色、脸型等分析信息，实现推荐记录的完整详情展示。

### 4.5 联查逻辑说明

推荐详情查询是一个典型的**三表级联查询**，查询链路为：

```
recommendation → user（获取用户昵称）
        ↓
recommendation → analysisresult（获取肤色、脸型等分析数据）
```

通过两次左连接，将推荐记录与用户信息、分析结果信息聚合在一起，实现一站式的推荐详情查询。

## 五、考核常见问题及快速应对方案

### 5.1 简单修改类（1-3分钟）

#### 问题1：列表展示字段映射修改

**场景**：原来展示的用户ID，现在展示为昵称

**修改位置**：前端对应HTML文件

**修改方法**：
- 检查表格列定义，修改表头名称
- 检查 `renderTable` 函数中的字段映射

#### 问题2：电话号码脱敏

**场景**：屏蔽中间4位，如 `138****1234`

**修改位置**：前端 `common.js` 添加工具函数

**修改方法**：
```javascript
function maskPhone(phone) {
  if (!phone) return '-';
  return phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
}
```

#### 问题3：修改时某些字段不更新

**场景**：修改照片时，无论图片地址输入什么，都不修改数据库中的图片地址

**修改位置**：后端 `photoRouter.js` 的 `/photo/update` 接口

**修改方法**：
```javascript
let sql = 'UPDATE photo SET AnalysisStatus = ? WHERE PhotoID = ?';
DB.connect(sql, [Status, +PhotoID], ...)
```

#### 问题4：修改每页条数

**场景**：将每页显示10条改为每页显示20条

**修改位置**：前端各页面的 `pageSize` 变量

**修改方法**：
```javascript
const pageSize = 20;
```

#### 问题5：在列表上方显示总记录数

**修改位置**：前端各页面的表格上方

**修改方法**：
```html
<div style="margin-bottom:10px;color:var(--text-3);">
  共 <strong style="color:var(--primary);">${total}</strong> 条记录
</div>
```

### 5.2 复杂修改类（5-8分钟）

#### 问题1：给表加一个字段并完成CRUD

**场景**：给推荐记录表增加"推荐评分"字段

**操作步骤**：

1. **数据库新增字段**：
```sql
ALTER TABLE recommendation ADD COLUMN RecommendScore INT DEFAULT NULL;
```

2. **后端修改**（`recommendRouter.js`）：
   - `/recommend/page` 查询语句添加字段
   - `/recommend/add` 插入语句添加字段
   - `/recommend/update` 更新语句添加字段

3. **前端修改**（`recommend.html`）：
   - 表格添加评分列
   - 新增/修改弹窗添加评分输入框

#### 问题2：多条件检索复杂查询

**场景**：在分析结果查询中增加"体型"筛选条件

**修改位置**：后端 `analysisRouter.js` + 前端 `analysis.html`

**修改方法**：

后端：
```javascript
if (bodyShape) { whereSql += ' AND a.BodyShape = ?'; params.push(bodyShape); }
```

前端：添加下拉框
```html
<select class="form-select" id="searchBodyShape">
  <option value="">全部</option>
  <option value="瘦">瘦</option>
  <option value="适中">适中</option>
  <option value="丰满">丰满</option>
</select>
```

#### 问题3：真删除变假删除

**场景**：照片删除改为软删除，增加 `IsDeleted` 字段

**操作步骤**：

1. **数据库修改**：
```sql
ALTER TABLE photo ADD COLUMN IsDeleted TINYINT DEFAULT 0;
```

2. **后端修改**（`photoRouter.js`）：
   - `/photo/page` 查询添加 `AND IsDeleted = 0`
   - `/photo/delete` 改为 `UPDATE photo SET IsDeleted = 1 WHERE PhotoID = ?`

#### 问题4：查询关键词对特定字段进行查询

**场景**：在照片管理中，输入关键词同时匹配用户昵称和图片地址

**修改位置**：后端 `photoRouter.js`

**修改方法**：
```javascript
if (keyword) { 
  whereSql += ' AND (u.NickName LIKE ? OR p.PhotoURL LIKE ?)'; 
  params.push(`%${keyword}%`, `%${keyword}%`); 
}
```
